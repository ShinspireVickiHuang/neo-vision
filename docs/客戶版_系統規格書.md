## 1. 文件目的
本文件用於向客戶說明「表單查詢系統（Prototype）」的功能範圍、使用方式、畫面欄位意義與查詢規則，作為需求確認與驗收依據。

## 2. 系統概述
### 2.1 系統提供什麼
- 提供一個查詢頁，讓使用者依條件篩選表單案件。
- 以列表方式呈現查詢結果，並提供「開啟」與「匯出」入口。

### 2.2 Prototype 限制（本版不包含）
- 本 Prototype 以前端假資料展示，不連接後端資料庫。
- 不包含：登入權限、分頁/排序、表單詳細頁。
- 匯出為前端下載 CSV（範圍/欄位見第 8 章），非後端報表。
- 「開啟 / 申請」目前以提示視窗代表（後續可串接真實功能）。

## 3. 使用者角色與情境（簡版）
- 一般使用者：需要查詢自己送出的申請單（或查看處理中案件）。
- 承辦/行政人員：需要依送單人快速定位案件（本 Prototype 用「處理種類」切換顯示送單人欄位）。

> 註：角色權限與可見資料範圍，若未來要做真實系統，需另行定義與串接。

## 6. 查詢條件（欄位對照與規則）

### 6.1 表單名稱（下拉）
- 用途：限制只查某一類表單。
- 選項來源：由系統資料中的「表單名稱」自動整理（去重後列出）。
- 操作：選取改變即更新結果。
- 規則：若未選表單名稱，查詢結果固定為 0 筆。

### 6.2 案場（已整合進查看對象）
（已整合進「查看對象」；不再提供獨立的案場下拉）

### 6.2 表單單號（文字輸入）
- 用途：用關鍵字查表單單號。
- 操作：輸入即時更新結果。
- 比對方式：包含比對。

### 6.3 查看對象（案場 / 組織）
- 用途：決定查詢範圍。
- 兩種模式：
	- 案場：可選特定案場，或選「不限」表示不限制案場。
	- 組織：可選部門或人員；並提供「選擇自己」。
- 預設：組織 + 自己。

### 6.4 處理種類（下拉）
- 選項：不限 / 本人申請表單 / 本人承辦表單
- 用途：切換是否需要輸入「送單人」。
- 規則：此欄位本身不直接影響查詢結果；只影響「送單人」欄位顯示/隱藏。
- 顯示條件：僅在「查看對象 = 組織」且選取「自己」時顯示。

### 6.5 送單人（文字輸入）
- 顯示條件：僅在「處理種類 = 本人承辦表單」時顯示。
- 用途：用關鍵字查送單人。
- 比對方式：包含比對（例如輸入「李」可匹配「李俐君」）。
- 操作：輸入即時更新結果。

### 6.6 申請日期（起迄）
- 用途：限制申請時間落在指定日期區間。
- 操作：起日或迄日任一變更即更新結果。
- 規則：以每筆案件的「申請時間」日期部分進行區間篩選。

### 6.7 狀態（Tabs）
- 位置：結果區上方的狀態 Tabs。
- 規則：結果依狀態分組為 Tabs，切換 Tab 會更新表格與筆數。
- Tab 順序：審批中 / 被取消 / 被暫停 / 已結案（已結案在最後）。
- 筆數顯示：除「已結案」外，其餘 Tab 會顯示筆數。

## 7. 查詢結果列表（欄位定義）

### 7.1 顯示規則
- 顯示筆數：`查詢結果：共 N 筆資料`
- 預設（剛開啟頁面/未選表單）：顯示 0 筆，表格不顯示。
- 若 N = 0：只顯示筆數，表格不顯示。
- 若 N > 0：顯示表格。

> 註：N 為目前狀態 Tab 下的筆數。

### 7.2 表格欄位（會依表單類型切換）
- 當表單為「人事｜試用員工考核表」：案場 /（組織＊）/ 員工姓名 / 現職單位 / 狀態 / 送單人 / 申請日期 / 目前關卡 / 操作
- 當表單為「會議記錄」：週會/月會 /（組織＊）/ 時間 / 地點 / 主席 / 紀錄 / 狀態 / 送單人 / 申請日期 / 目前關卡 / 操作
- 其他表單（預設）：案場 /（組織＊）/ 申請人 / 狀態 / 送單人 / 申請日期 / 目前關卡 / 操作

> 註：＊「組織」欄位僅在「查看對象 = 組織」時顯示。

## 8. 操作功能（Prototype 入口）
### 8.1 開啟
- 位置：結果列表每一列「開啟」按鈕
- Prototype 行為：跳出提示視窗，表示可開啟該筆案件

### 8.2 匯出
- 位置：結果區右上角「匯出」按鈕
- Prototype 行為：下載 CSV 檔（匯出「目前狀態 Tab」的查詢結果）
- 匯出欄位：依表單類型輸出核心欄位，且不包含：狀態 / 送單人 / 申請日期 / 目前關卡 / 操作

### 8.3 申請
- 位置：查詢條件區右下角「申請」按鈕
- Prototype 行為：跳出提示視窗，表示可進入申請流程

## 9. 查詢規則（較技術但可驗收）
系統會依下列順序套用條件（有填才生效）：
1) 表單名稱（必選；未選直接顯示 0 筆）  
2) 查看對象（案場/組織；案場不限則不限制）  
3) 表單單號（包含比對）  
4) 送單人（包含比對）  
5) 申請日期區間  

> 註：狀態不作為查詢條件欄位；結果依狀態分組為 Tabs。

## 10. 驗收清單（客戶可直接勾選）
- 開啟頁面後，預設顯示 0 筆且表格不顯示。
- 未選表單名稱時，結果固定為 0 筆。
- 任一查詢條件變更後，列表會立即更新。
- 狀態 Tabs 順序正確，且已結案不顯示筆數。
- 處理種類僅在「組織 + 自己」時出現；切到「本人承辦表單」時會出現「送單人」欄位；切回其他選項會隱藏且清空。
- 查詢結果為 0 筆時，表格不顯示但筆數仍顯示為 0。
- 點擊「匯出」會下載 CSV（目前狀態 Tab 的結果）。
- 點擊任一列「開啟」會有明確反應（Prototype 提示）。
- 點擊「申請」會有明確反應（Prototype 提示）。
