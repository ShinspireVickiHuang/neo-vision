更新日期：2026/03/10
主要畫面：`index.html`

## 1. 目標與定位
- 本文件用於定義「表單查詢 / 結果列表」Prototype 的使用者操作流程與互動規則，作為 PM/使用者/開發對齊依據。
- 本 Prototype 為純前端展示用（假資料），重點在「查詢條件 → 即時篩選 → 列表呈現 → 開啟/匯出入口」的互動一致性。

## 2. 範圍（Scope）
### 2.1 內含
- 查詢條件區（表單名稱、表單單號、查看對象、申請日期區間、處理種類/送單人〔條件顯示〕）
- 查詢結果列表（筆數、表格、操作按鈕）
- 匯出（下載 CSV：匯出目前狀態 Tab 的查詢結果）
- 開啟/申請入口（Prototype 以提示視窗代表）

### 2.2 不含（Non-goals）
- 不實作真正後端查詢、權限、分頁、排序、表單詳細頁跳轉
- 不處理跨瀏覽器一致性以外的複雜相容性議題

## 3. 使用者流程（User Flow）
1) 使用者開啟頁面 → 預設顯示 0 筆（需先選表單名稱）
2) 使用者調整任一查詢條件 → 立即更新列表與筆數
3) 使用者點「匯出」 → 下載 CSV（匯出「目前狀態 Tab」的查詢結果）
3.5) 使用者點「申請」 → 觸發申請入口（Prototype 提示）
4) 使用者點某列「開啟」 → 觸發開啟表單入口（Prototype 提示）

## 4. 畫面與欄位規格（index.html）

### 4.1 查詢條件區

#### A. 表單名稱（下拉）
- 標籤：表單名稱
- 來源：由資料中的 `formName` 去重後產生選項
- 預設值：空值（顯示「請選擇表單」）
- 互動：變更時立即執行查詢
- 規則：未選表單時，查詢結果固定為 0 筆

#### B. 表單單號（文字輸入）
- 標籤：表單單號
- 互動：輸入即時查詢（on input）
- 比對方式：包含比對（contains）

#### C. 查看對象（案場 / 組織）
- 標籤：查看對象
- 組成：
  - 下拉：案場 / 組織
  - 只讀輸入框：顯示目前選取（例如「不限 / 自己 / 某部門」）
  - 按鈕「選擇」：開啟彈窗選取
- 預設：組織 + 自己
- 案場模式：彈窗顯示案場清單，並提供「不限」
  - 若選「不限」：不限制案場（可查該表單全部資料）
- 組織模式：彈窗顯示樹狀結構（部門/人員）並提供「選擇自己」
  - 選人員：以「申請人/送單人/目前關卡文字包含」判斷資料歸屬
  - 選部門：以資料的 `cDeptId` 命中（必要時以部門成員名單輔助）

#### D. 處理種類（下拉）
- 標籤：處理種類
- 選項：
  - 不限
  - 本人申請表單
  - 本人承辦表單
- 主要影響：只控制「送單人」欄位是否顯示
- 顯示條件：僅在「查看對象 = 組織」且目前選取為「自己」時顯示
- 互動：變更時依序執行：
  1) 切換送單人欄位顯示
  2) 立即執行查詢

#### E. 送單人（文字輸入）
- 標籤：送單人
- 顯示條件：僅在「處理種類 = 本人承辦表單」時顯示
- 互動：輸入即時查詢（on input）
- 比對方式：包含比對（contains）

#### F. 申請日期（起迄）
- 欄位：起日、迄日（日期選擇）
- 互動：任一日期變更即觸發查詢
- 比對基準：以每筆資料的 `applyTime` 日期部分（YYYY/MM/DD）進行區間篩選
- 注意（規格提醒）：日期欄位輸入為 YYYY-MM-DD，資料為 YYYY/MM/DD；實作時需先統一格式再比大小，避免直接字串比較造成錯誤

### 4.2 結果區

#### A. 筆數顯示
- 文案格式：查詢結果：共 N 筆資料
- N 的定義：目前「狀態 Tab」下的筆數（而非全部狀態合計）

#### B. 狀態 Tabs（依狀態切換）
- Tab 順序：審批中 / 被取消 / 被暫停 / 已結案（已結案在最後）
- Tab 顯示筆數：
  - 審批中/被取消/被暫停：顯示（N）
  - 已結案：不顯示筆數
- 切換 Tab：更新表格與筆數

#### C. 表格顯示/隱藏
- 若結果 = 0 筆：隱藏表格（只顯示筆數）
- 若結果 > 0 筆：顯示表格

#### D. 表格欄位（會依表單類型切換）
- 當表單為 `人事｜試用員工考核表`
  - 案場 /（組織＊）/ 員工姓名 / 現職單位 / 狀態 / 送單人 / 申請日期 / 目前關卡 / 操作
- 當表單為 `會議記錄`
  - 週會/月會 /（組織＊）/ 時間 / 地點 / 主席 / 紀錄 / 狀態 / 送單人 / 申請日期 / 目前關卡 / 操作
- 其他表單（預設）
  - 案場 /（組織＊）/ 申請人 / 狀態 / 送單人 / 申請日期 / 目前關卡 / 操作

> 註：＊「組織」欄位僅在「查看對象 = 組織」時顯示。

#### E. 操作
- 每列提供「開啟」按鈕
- 點擊後：觸發「開啟表單」入口（Prototype 以提示視窗代表）

#### F. 匯出
- 匯出範圍：僅匯出「目前狀態 Tab」下的查詢結果
- 匯出格式：下載 CSV（含 BOM，便於 Excel 開啟）
- 匯出欄位：依表單類型輸出核心欄位（且不包含：狀態 / 送單人 / 申請日期 / 目前關卡 / 操作）

#### G. 申請
- 點擊「申請」：觸發申請入口（Prototype 以提示視窗代表）

## 5. 查詢邏輯（Business Rules）
- 查詢會依序套用以下條件（有填才生效）：
  1) 表單名稱（必選；未選直接顯示 0 筆）
  2) 查看對象（案場/組織；案場不限則不限制）
  3) 表單單號（包含比對）
  4) 送單人（包含比對）
  5) 申請日期區間（起迄）
- 狀態不作為查詢條件欄位；結果會依狀態分組為 Tabs。
- 「處理種類」不直接參與資料篩選，只影響送單人欄位顯示。

## 6. 假資料欄位（用於 Prototype）
每筆資料至少包含：
- 表單名稱 `formName`
- 案場 `site`
- 狀態 `status`
- 送單人 `applicant`
- 申請時間 `applyTime`
- 目前關卡 `currentStage`

## 7. 驗收條件（Acceptance Criteria）
- 開啟頁面時：列表預設顯示 0 筆，筆數顯示為 0 且表格隱藏。
- 未選表單名稱時：不進行其他篩選，結果維持 0 筆。
- 任何一個條件改變（下拉/日期/輸入）時：列表立即更新且筆數正確。
- 狀態 Tabs 順序正確，且已結案不顯示筆數。
- 處理種類在「組織 + 自己」時才出現；切到「本人承辦表單」時顯示送單人欄位；切回其他選項時隱藏且清空。
- 結果為 0 筆時：表格不可見（僅保留筆數顯示）。
- 點擊匯出：下載 CSV，且內容為目前狀態 Tab 的結果。
- 點擊申請/開啟：能明確觸發對應入口（Prototype 可用提示視窗表示）。
